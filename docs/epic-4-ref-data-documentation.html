Why Reference Data is the Engine's Critical Path

A property address alone is not enough to determine loan eligibility. Every agency program has geographic constraints that are evaluated before any borrower qualification. Getting these wrong results in loans that fail in underwriting — or worse, fund incorrectly.

ProgramGeographic CheckIncome CheckData Source
FHA Loan amount ≤ county FHA limit (1–4 unit) None HUD FHA Mortgage Limits
VA Limit applies only with reduced entitlement None FHFA Conforming Limits
Conventional Loan ≤ conforming limit; high-balance flag None FHFA Conforming Limits
USDA SFGH Property in eligible rural area (shapefile) Household ≤ 115% AMI (all adults) USDA RD shapefile + income limits
HomeReady / Home Possible None (any conforming county) Borrower ≤ 80% AMI; waived in low-income tracts HUD/FFIEC AMI tract data

All five checks key off FIPS code (county-level) and/or census tract GEOID (11-digit). ZIP codes are never used for eligibility — they are non-jurisdictional and can span county lines.

4.5 FHA Loan Limits HUD Annual
SourceHUD FHA Mortgage Limits database URLhuduser.gov/portal/datasets/fha/fha_limits/ FormatCSV download, also HUD API UpdatedAnnually in November; effective January 1 Key5-digit county FIPS + effective year

The FHA floor and ceiling are set at 65% and 150% of the national conforming limit respectively. Every county receives a limit between these bounds based on local median home prices.

Limit Type1-Unit (2025)Applies To
Floor (FLOOR)$524,225Lower-cost counties — national minimum

HomeReady / Home Possible income limit waiver: If the census tract median income ≤ 80% of the MSA median income, the tract is classified as a "low-income tract" and there is no income limit for HomeReady/HomePossible borrowers in that tract. Both FNMA and FHLMC use identical rules per their voluntary alignment.

4.11–4.12 CBSA/MSA Crosswalk + Texas HOI Premiums Supplemental

CBSA/MSA Crosswalk (4.11): Maps county FIPS to CBSA code, MSA status, and metropolitan/micropolitan designation. Used for USDA rural determination (populations in MSAs above 35,000 are presumptively ineligible) and USDA income limit lookups (limits are per-county or per-MSA).

Texas HOI Premiums (4.12): Average annual homeowner's insurance premium by ZIP code for Texas properties. Keyed by ZIP (not FIPS) because HOI is an actuarial estimate, not a regulatory fee tied to a specific jurisdiction. Only Texas initially; other states use a national estimator.

USDA Rural Eligibility — Deep Dive

The USDA check is the most complex geographic eligibility check in the engine. It requires understanding three separate data sources and how they interact.

The Three USDA Data Sources

1. Ineligible Area Shapefile (primary — Task 4.7)

The source of truth. 2,607 polygons covering urban and suburban areas. A property is USDA-eligible if its coordinates fall outside all polygons.

ArcGIS layer: layerID = 4 → "RHS_SFH_MFH_IELG_2011" (Rural Housing Service, Single Family Housing / Multi-Family Housing, Ineligible, 2011 vintage, 2018-08-23 update)

2. MFH by Tract CSV (supplemental — Task 4.8)

7,303 census tracts where USDA has active multi-family housing projects. These tracts are confirmed rural by USDA's own project approval history. Used to resolve edge cases at shapefile polygon boundaries.

3. USDA Income Limits (gate — Task 4.9)

Even when a property passes the geographic eligibility check, the household income must be ≤ 115% of AMI. The income limits are per-county, per-family-size (8 sizes), annually updated.

Pre-computation Strategy (Task 4.7)

Rather than running a live PostGIS query for every property, the engine pre-computes eligibility for all ~74,000 2020 Census tracts and stores results in usda_rural_eligibility. The pre-computation pipeline:

  1. Load SFH_MFH_Ineligible.shp into PostGIS
  2. Download TIGER/Line tract boundaries for all 50 states + DC
  3. For each tract: check if centroid falls within any ineligible polygon → is_sfh_eligible
  4. Compute pct_eligible: % of tract area outside ineligible zones (for boundary tracts)
  5. Cross-reference with MFH CSV: tracts with USDA projects that appear ineligible are flagged for manual review
Production vs testPre-computation runs once when new shapefile data is released (roughly every 10 years + interim updates). The JsonFileStore for tests contains the pre-computed results for the 5 fixture properties. The production PostgresStore holds all ~74,000 tracts.

The Complete USDA Eligibility Check Sequence

1
Property coordinates available?
FCC geocoder must have run (Epic 5). Without tract_geoid, all USDA checks return conservative: ineligible.
2
Geographic eligibility check
usda_rural_eligibility.is_sfh_eligible WHERE geoid = tract_geoid. If false → USDA ineligible; stop here.
3
Income limit check
annual_household_income ≤ usda_income_limits.limit_size_{household_size}. Uses ALL adults in home, including non-borrowers. If over limit → USDA ineligible.
4
Property type check
Only 1–2 unit residential properties. Must be owner-occupied primary residence. No investment properties, no mobile homes on leased land.
5
USDA eligible ✓
Proceed to USDA-specific pricing (guarantee fee, annual fee).

Versioning System

Defined in src/versioning.rs. Every piece of reference data is wrapped in Versioned<T>. Every analysis stores a DataVersionManifest.

Why this mattersCFPB examination requires perfect replay of any historical loan analysis. If LLPA matrices changed between when a loan was quoted and when it funded, the examiner needs to see exactly which matrix was in effect. The manifest + engine git SHA enables exact reproduction of any analysis.

VersionId

Format: "{table}:{effective_date}"

let v = VersionId::new("fha_loan_limits", NaiveDate::from_ymd_opt(2025, 1, 1).unwrap());
assert_eq!(v.as_str(), "fha_loan_limits:2025-01-01");

Versioned<T>

pub struct Versioned<T> {
    pub version_id:     VersionId,
    pub effective_date: NaiveDate,
    pub data:           T,          // FhaLoanLimits, GseLoanLimits, etc.
}

RefDataStore Trait

Defined in src/store/mod.rs. Every data access goes through this interface — the engine never talks to a backend directly.

pub trait RefDataStore: Send + Sync {
    fn fha_loan_limits(&self, fips_code: &str, year: u16)
        -> RefDataResult<Versioned<FhaLoanLimits>>;

    fn gse_loan_limits(&self, fips_code: &str, year: u16)
        -> RefDataResult<Versioned<GseLoanLimits>>;

    fn usda_rural_eligibility(&self, geoid: &str)
        -> RefDataResult<Option<UsdaruralEligibility>>;

    fn usda_income_limits(&self, fips_code: &str, effective_date: NaiveDate)
        -> RefDataResult<Versioned<UsdaIncomeLimit>>;

    fn usda_mfh_by_tract(&self, geoid: &str)
        -> RefDataResult<Option<UsdaMfhByTract>>;

    fn ami_tract_data(&self, geoid: &str, year: u16)
        -> RefDataResult<Option<Versioned<AmiTractData>>>;

    // ── Single-call unified API ──────────────────────────────────────────
    fn geo_eligibility(&self, fips_code: &str, tract_geoid: Option<&str>, year: u16)
        -> RefDataResult<GeoEligibility>;

    fn current_version(&self, dataset: &str)
        -> RefDataResult<VersionId>;
}

Semantic Contract

  • All methods return the data as-of the requested year, using the most recent version available that is ≤ the requested year.
  • Methods return Err(NotFound) when no data exists for that FIPS/year — callers must handle this explicitly.
  • usda_rural_eligibility and usda_mfh_by_tract return Option — a missing tract record is conservative-ineligible, not an error.
  • geo_eligibility assembles all sources in one call; implementations are expected to batch the underlying lookups.

GeoEligibility API

The single call that powers all geographic eligibility decisions for a property. Epic 7 (eligibility engine) calls this once per scenario and runs all program checks from the result.

Query Pattern

// Called once when a scenario is created:
let geo = store.geo_eligibility(
    "48209",              // county FIPS from FCC resolution or TaxTract field
    Some("48209010905"),  // census tract GEOID from FCC Block.FIPS[0..11]
    2025,                 // effective year for loan limits
)?;

// Then consumed by the eligibility engine:
let fha_ok   = geo.fha_loan_within_limit(adjusted_loan, 1);
let (conv_ok, is_hb) = geo.gse_loan_status(loan_amount, 1);
let usda_geo = geo.usda_sfh_eligible;
let usda_inc = geo.usda_income_eligible(household_income, 3);
let hp_ok    = geo.hp_income_eligible(borrower_income);

Complete Eligibility Check Sequence

1
FHA VA Conventional — Loan Amount vs County Limit
FHA: adjusted_loan ≤ fha_limit_{n}_unit (UFMIP-financed amount)
Conventional: loan ≤ gse_limit_{n}_unit → if between standard & county limit → high-balance flag
VA: Skip if full entitlement; else check loan ≤ gse_limit_1_unit
2
USDA — Geographic Rural Eligibility
Check usda_sfh_eligible = true. If false → USDA ineligible, skip all remaining USDA checks. Requires tract_geoid from FCC resolution.
3
USDA — Household Income Check
Check annual_household_income ≤ usda_income_limits[household_size-1]. Household income = ALL adults in home, including non-borrowers.
4
HomeReady / Home Possible — AMI Income Check
If hp_income_limit_waived = true (low-income tract) → no limit. Otherwise: borrower_income ≤ ami_80pct.
5
Conventional — High-Balance Designation
Set is_high_balance = loan > STANDARD_1_UNIT_2025 AND gse_is_high_cost. Triggers high-balance LLPA tiers in Epic 10.
